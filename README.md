# Profiler

Windows용 초경량 구간 성능 측정 라이브러리. `QueryPerformanceCounter` 기반으로 코드 구간의 실행 시간을 마이크로초(㎲) 단위로 집계해 텍스트 보고서로 출력한다.

## 사용법

`Profile.h`와 `Profile.cpp` 두 파일을 프로젝트에 추가하기만 하면 된다 (드롭인 방식).

```cpp
#include "Profile.h"

for (int i = 0; i < 10; i++)
{
    PRO_BEGIN("Update");
    Update();
    PRO_END("Update");
}
PRO_PRINT();   // profile.txt로 보고서 출력 (생략해도 정상 종료 시 자동 출력됨)
```

| 매크로 | 동작 |
|---|---|
| `PRO_BEGIN(tag)` | 측정 구간 시작 |
| `PRO_END(tag)` | 측정 구간 종료, 통계에 반영 |
| `PRO_PRINT()` | `profile.txt`로 보고서 출력 (`ProfileDataOutText("파일명")`으로 경로 지정 가능) |
| `PRO_RESET()` | 모든 측정치·오류 기록 초기화 |

프로젝트 전처리기 정의에 `PROFILE_DISABLE`을 추가하면 매크로가 전부 빈 코드로 치환되어 측정 오버헤드가 0이 된다.

## 출력 예시

```
-------------------------------------------------------------------------------------------------------------------
 ThreadId | Name                                    |    Average(us) |        Min(us) |        Max(us) |       Call
-------------------------------------------------------------------------------------------------------------------
    21532 | hi                                      |     30144.6167 |     29926.9000 |     31068.9000 |         10
    21532 | wawa                                    |     15195.9833 |     14442.5000 |     15871.2000 |         10
-------------------------------------------------------------------------------------------------------------------
```

- **Average**: 호출이 5회 이상이면 최댓값 2개·최솟값 2개를 제외한 평균(이상치 배제), 5회 미만이면 단순 평균.
- **Min/Max**: 관측된 최솟값/최댓값.
- **Call**: 완료된(BEGIN~END 짝이 맞은) 측정 횟수. `END`를 받지 못한 태그(짝 없는 BEGIN)나 첫 측정이 아직 끝나지 않은 태그는 **Call=0** 행으로 표시되어 누락이 눈에 띈다.

## 동작 방식과 제한

- **스레드 모델**: 측정치는 스레드별로 분리 집계된다 (`thread_local` 저장소, 핫패스 무경합 — 출력 시에도 슬롯을 잠깐 복사만 하고 파일 I/O는 락 밖에서 수행하므로 측정 스레드가 디스크 쓰기에 멈추지 않는다). 여러 스레드가 같은 태그를 써도 안전하며, 보고서에는 스레드 ID별로 행이 나뉘어 출력된다. 스레드가 종료해도 그 스레드의 측정치는 보고서에 남는다.
- **용량**: 스레드당 태그 32개, 태그 이름 최대 39자(초과분은 잘림). 태그는 잘린 길이로 비교·표시되므로, 앞 39자가 같은 서로 다른 태그는 같은 태그로 취급된다.
- **오류 처리**: 짝 없는 `END`(BEGIN 없는 END), 연속 BEGIN, 용량 초과 등의 **오사용**은 프로세스를 중단시키지 않고 보고서 하단 `[Errors]` 섹션에 (발생 스레드 ID와 함께) 기록된다. 기록 한도(32건)를 넘으면 초과 건수가 안내된다. 디버거가 붙어 있을 때만 `DebugBreak()`로 알리며, 그때 디버거가 분리되어도 프로세스는 죽지 않는다. (짝 없는 BEGIN은 `[Errors]`가 아니라 위의 Call=0 행으로 드러난다.)
- **연속 BEGIN**: 같은 태그에 END 없이 BEGIN을 다시 호출하면 이전 구간은 버려지고 새로 시작한다 (오류로 기록됨).
- **출력 직렬화**: 자동 출력과 여러 스레드의 `PRO_PRINT()`가 동시에 일어나도 내부적으로 직렬화되어 리포트가 유실되지 않는다.
- **주의**:
  - 비정상 종료(크래시, `TerminateProcess`) 시에는 자동 출력이 실행되지 않으므로, 중요한 시점마다 `PRO_PRINT()`를 직접 호출해 중간 결과를 남기는 것을 권장한다.
  - 다른 정적 객체의 소멸자에서 수행되는 측정은 링크 순서에 따라 자동 출력 이후에 실행될 수 있어 보고서에서 빠질 수 있다. 종료 구간을 측정하려면 해당 지점에서 명시적으로 `PRO_PRINT()`를 호출할 것.
  - `PRO_RESET()`은 진행 중인 측정이 없을 때 호출할 것.
  - 전역 메모리 훅(`operator new`/`malloc` 후킹) **내부**에서는 `PRO_` 매크로를 사용하지 말 것(첫 호출 시 저장소 초기화가 할당을 유발해 재진입한다 — 라이브러리는 이 경우 해당 호출을 안전하게 건너뛴다).

## 빌드

Visual Studio 2022 (v143), C++17, `/W4`, `/utf-8`. `Profiler.sln`을 열고 빌드하면 샘플(`Profiler.cpp`)이 함께 빌드된다.
